Profile

I am a PhD candidate in the Department of Computer Science and Engineering at the University of Ioannina in Greece.

I am Giorgos E. Kappes and I was born in Ioannina, Greece in 1987. I graduated the 2nd High School of Ioannina in 2004 and obtained my BSc degree from the Department of Computer Science, of the University of Ioannina in 2011.
During my undergraduate studies, I gained basic knowledge in building high performance systems, software and Internet services, as well as in evaluating and improving existing ones. Furthermore, I had the opportunity to work on a thesis entitled “Logging file access patterns for a more efficient file search on file systems” which helped me to deepen my knowledge in the fields of information retrieval and file and storage systems. I completed my MSc in the Systems Research Group (SRG) of the same department under the supervision of Prof. Stergios Anastasiadis. In our research we examined approaches for efficient and effective support of multitenancy in filesystems used by virtual machines. The title of my MSc thesis was “Scalable Access Control for Secure Multi-Tenant Filesystems”.

Currently, I am a PhD candidate at the same department and a member of Systems Research Group (SRG).
My main research interests include some hot topics of computer systems, as: Virtualization, File and Storage Systems, and Security of Computer Systems.

Research

My main research interests lie in the fields of virtualization, file and storage systems, as well as security and privacy. However, I am interested in pretty much everything to do with computer systems. Following are some of the projects that I have worked on:

Efficient, effective, and secure multitenancy. Secure access control is a challenging problem that organizations face in collaborative virtual environments, which has prevented many of them from migrating critical data or applications into these environments. In our research we examined approaches for efficient and effective support of multitenancy in filesystems used by virtual machines. We developed a Linux-based prototype solution over Ceph and obtained encouraging experimental results about the scalability and low overhead of our design. (MSc Thesis. Supervisor: Stergios V. Anastasiadis)

Feeding a desktop search engine with right results. The existing desktop search systems use web-based technologies, which are not optimal for desktop environments. To address this problem, some advanced desktop search systems create virtual links between files using methods that can generate too many false connections between files and also lead to performance bottlenecks due to the file connections that they maintain in a graph structure. In order to feed the search engine with the right results we developed a prototype filter driver over Windows NTFS to record important access characteristics of the local files. (BSc Thesis. In collaboration with Eirini Micheli. Supervisor: Stergios V. Anastasiadis)

OFIDIA - Operational Fire Danger Prevention Platform The main objective of the project is to build a cross-border operational fire danger prevention infrastructure that advances the ability of regional stakeholders across Apulia and Ioannina Regions to detect and fight forest wildfires through more effective patrolling, environmental monitoring, weather modeling, fire danger forecasting and automated access to related historical data.
My assigned task was the design of storage models for climate-change data. In particular, the determination of models for storing and processing scientific data on multicore systems and virtual machines. I also contributed on the deployment and setup of the platform's infrasturcture. (Project website)

Technical Reports

  • Giorgos Kappes, Andromachi Hatzieleftheriou, Stergios V. Anastasiadis, Dike: Virtualization-aware Access Control for Multitenant Filesystems. Technical Report DCS2013-1, Department of Computer Science, University of Ioannina, February 2013. (Paper).

    In a virtualization environment that serves multiple customers (or tenants), storage consolidation at the filesystem level is desirable because it enables data sharing, administration efficiency, and performance optimization. Today the scalable deployment of filesystems in such environments is challenging due to intermediate translation layers required for purposes of networked file access or identity management. First we analyze the security requirements in multi-tenant filesystems. Then we introduce the Dike authorization architecture, which combines native access control with tenant namespace isolation that is backwards compatible to object-based filesystems. We experimentally evaluate a prototype implementation that we developed, and show that our solution incurs limited added performance overhead.

Tutorials

Here you can find a list of tutorials I have written on various topics to help me get up to speed on them. I am making these tutorials available here, in case they might be helpful to others.

Virtualization

Installing and Using Xen. Xen is an Open-Source hypervisor that creates and manages domains for multiple virtual machines. It uses paravirtualization, where the guest operating systems are aware that they are running in a virtualized environment. This guide will show you how to install xen from source and how to create guest domains. Furthermore it includes topics like network configuration, migration, root on NFS and CPU management [pdf].
Below are included some domU configuration templates:

  1. A domU configuration template
  2. A domU configuration template with NFS root
  3. An HVM domU configuration template

User Mode Linux Installation Guide. In this guide we show you how to build UML from source and how to configure it. In this context we try to point out how to create and configure a file system for the UML guest and how to provide networking to it [pdf].

Root over NFS on User Mode Linux. The root file system of a UML guest can be stored on a disk image in the host system, on a disk partition of the host system or in a directory hierarchy in the host system which can be exported to the client via humfs or NFS. This guide follows the last approach. More specifically, the root file system for the UML guest will be located on an exported directory hierarchy stored in a NFS server. This server could be the host system, a remote machine or even another UML guest. Our goal is the UML client to mount this exported directory hierarchy as its root file system via NFS [pdf].

File Systems

NFS v4 Installation Guide. In this guide we will try to explain how to build and setup an NFSv4 server and how to configure it in order to export a file system for the clients. Furthermore we will see how to setup an NFS client and how to mount an exported file system [pdf].
We have included below some useful configuration files and init scripts:

  1. nfs-kernel-server.default
  2. nfs-common.default
  3. nfs-kernel-server.init
  4. nfs-common.init
  5. idmapd.conf
  6. exports

Linux and Networking

ssh. A step by step guide to teach you how to use ssh. ssh (Secure Shell) is a secure network protocol that allows data transfer between two computers. The ssh not only encrypts the data exchanged during the session, but it offers a secure identification system and other features such as secure file transfer [pdf in Greek].

nmap. A step by step guide to teach you how to use nmap. nmap is a network analyzer. It was designed to monitor large networks, but works perfectly against a single host [pdf in Greek].

aircrack-ng. In this tutorial we will show you how to use the aircrack-ng suite in order to audit your wireless network. The tutorial begins with a brief introduction to the basic principles of 802.11 networks and continues with an analysis of the WEP algorithm and its several serious weaknesses which were identified by cryptanalysts [pdf in Greek].

Programming

MPI's one-sided communication model. In this guide we briefly describe the main features of MPI's one-sided communications. They are designed to enable easy and efficient remote memory access and to provide higher performance in comparison to the model of point-to-point communications [pdf in Greek].

RAW Sockets. In this tutorial we briefly describe the Raw Sockets API. The Raw Sockets allow us to access and modify the header of a network packet. Raw sockets can be used to hide or spoof the attacker's source IP address during a Denial of Service (DoS) Attack [pdf in Greek].
